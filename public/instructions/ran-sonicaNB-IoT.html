Sonica: Software-defined Open NB-IoT Research Platform

Introduction

Notice: This is an archive repo that contains a basic feature set. To access the latest release with the most recent updates and full functionality, please visit here.

Sonica provides an open-source NB-IoT testbed that can communicate with commodity off-the-shelf (COTS) devices. It implements the NB-IoT version of eNodeB and an EPC with Cellular Internet-of-Things (CIoT) extensions. This makes it an ideal platform for quick prototyping and/or verification across various layers of the NB-IoT network.

Sonica Components

Sonica is designed to run in a Linux environment. To set up an NB-IoT network that can communicate with COTS hardware, SDR RF hardware is required. The currently tested hardware includes:

  • USRP B210
  • USRP X300

Before building Sonica, you need to install the required dependencies. For example, on Ubuntu, run the following command:

$ sudo apt-get install build-essential cmake libfftw3-dev \
   libmbedtls-dev libboost-program-options-dev libconfig++-dev \
   libsctp-dev

Build Setup

Besides the dependencies, you also need to install the following packages:

  • UHD: For communication with USRP
  • ZeroMQ: For RF emulation

Sonica's build system is based on Meson. To build the system, execute the following commands:

meson builddir
                 cd builddir
                 meson compile

Once built, the two binary programs will be located in the builddir directory:

  • eNodeB: sonica_enb/sonica_enb
  • EPC: third_party/srsepc_ciot/src/srsepc_ciot
