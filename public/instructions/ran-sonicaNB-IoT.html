Introduction

Notice: This is an archive repo with a basic feature set. To access the latest release with the most recent updates and full functionality, please visit here.

Sonica provides an open-source NB-IoT testbed that can communicate with commodity off-the-shelf (COTS) devices. It implements the NB-IoT version of eNodeB and an EPC with Cellular Internet-of-Things (CIoT) extensions. Thus, it provides a platform for quick prototyping and/or verification on various layers of the NB-IoT network.

Sonica Components

Sonica runs on a Linux environment. To set up an NB-IoT network that can communicate with COTS hardware, SDR RF hardware is necessary. The currently tested hardware includes USRP B210 and X300.

System Setup

Before building, you need to install dependencies packages. For example, on Ubuntu, use the following command: 

$ sudo apt-get install build-essential cmake libfftw3-dev \
   libmbedtls-dev libboost-program-options-dev libconfig++-dev \
   libsctp-dev

Additionally, UHD (for communication with USRP) and ZeroMQ (for RF emulation) are required.

Sonica's build system is based on Meson. To build, execute the following commands: 

[sonica]$ meson builddir
[sonica]$ cd builddir
[sonica/builddir]$ meson compile

The two binary programs are located within the 'builddir' directory:

  • eNodeB: sonica_enb/sonica_enb
  • EPC: third_party/srsepc_ciot/src/srsepc_ciot
