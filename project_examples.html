Core Network

This section explains how the core network has been set up for this test bench, allowing the process to be reproducible and adaptable for future updates.

System Tweaking

For optimal performance, it’s necessary to install low-latency kernels in the Linux OS. This minimizes communication latency between the SDR and processors.

sudo apt-get install linux-image-lowlatency

After rebooting and selecting the low-latency kernel, set the processor to full performance mode with the following commands:

watch grep "cpu MHz" /proc/cpuinfo
sudo apt-get install i7z
sudo i7z
sudo apt-get install cpufrequtils
sudo nano /etc/default/cpufrequtils
sudo update-rc.d ondemand disable
echo performance | tee /sys/devices/system/cpu/cpu*/cpufreq/scaling_governor

Installing the Core Network

The Core Network used is OpenAirInterface’s (OAI) product, which is fully functional and works on Ubuntu 20.04 LTS.

Clone and build the OAI Core Network repository:

git clone --branch v2.0.1 https://gitlab.eurecom.fr/oai/cn5g/oai-cn5g-fed.git
cd oai-cn5g-fed
git checkout -f v2.0.1
./scripts/syncComponents.sh
git submodule deinit --all --force
git submodule init
git submodule update

Configuring the Core Network

Adjust configuration parameters to match those of the gNodeB. The main parameters include MCC, MNC, and TAC, which need to align between the Core Network and gNodeB.

oai-amf:
container_name: "oai-amf"
image: oaisoftwarealliance/oai-amf:v2.0.1
environment:
- TZ=Europe/Paris
- MCC=208
- MNC=36
- REGION_ID=31
- SERVED_GUAMI_MCC_0=208
- PLMN_SUPPORT_MNC=36
- PLMN_SUPPORT_TAC=1

Launching the Core Network

First, update Docker and set Google nameservers:

cd /usr/local/bin && sudo rm -rf docker-compose
sudo curl -SL https://github.com/docker/compose/releases/download/v2.1.1/docker-compose-linux-x86_64 -o /usr/local/bin/docker-compose
sudo chmod +x docker-compose
sudo nano /etc/resolv.conf (add nameserver 8.8.8.8)
sudo systemctl daemon-reload
sudo systemctl restart docker

Launch the Core Network:

sudo python3 core-network.py --type start-basic

gNodeB

Installing the gNodeB

Install the OpenAirInterface gNodeB, using Ubuntu 20.04 for best compatibility. Clone the repository:

git clone https://gitlab.eurecom.fr/oai/openairinterface5g.git ~/openairinterface5g
cd ~/openairinterface5g
git checkout develop

Build the required dependencies:

cd cmake_targets
sudo ./build_oai -I

Configuring the gNodeB

Configure the gNodeB to match the core network parameters, particularly MCC, MNC, TAC, and IP settings. Update the configuration file:

gNB_ID = 0xe00;
tracking_area_code = 1;
plmn_list = ({ mcc = 001; mnc = 01; mnc_length = 2; });
amf_ip_address = { ipv4 = "192.168.70.132"; };

Launching the gNodeB

Launch the gNodeB:

sudo ./nr-softmodem -O gnb.conf --gNBs.[0].min_rxtxtime 6 --sa -E

UE

OpenAirInterface UE

To build the UE, use the same process as the gNodeB:

cd ~/openairinterface5g
source oaienv
cd cmake_targets
./build_oai -w USRP --ninja --UE -C

Add the UE's identity to the Core Network:

imsi = "208970100001127";
key = "fec86ba6eb707ed08905757b1bb44b8f";

Launching the UE

Launch the UE with the following command:

sudo ./nr-uesoftmodem -r 51 --numerology 1 --band 78 -C 3309480000 --sa --norkmod
