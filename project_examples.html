Introduction to Open-Source 4G/5G Network Implementations

The development of 4G and 5G networks has transformed global communications, making connectivity faster, more efficient, and increasingly flexible. Traditional mobile network implementations require extensive infrastructure and proprietary hardware, creating significant entry barriers for researchers, developers, and small-scale operators. Open-source 4G/5G solutions, leveraging the flexibility of Software Defined Radio (SDR), provide an accessible, modular alternative that enables users to deploy, test, and innovate on cellular network architectures without large investments.

In this context, open-source projects like OpenAirInterface (OAI), srsRAN, Open5GS, and Sonica NBIoT have become essential tools. These projects provide a complete stack for building and testing LTE and NR (5G) networks, supporting both virtualized and physical environments. They empower users to explore everything from basic LTE deployments to cutting-edge 5G research. This page will guide you through the details of these popular platforms, outlining the necessary setup, main features, and potential applications of each.

OpenAirInterface (OAI)

Project Overview: OpenAirInterface (OAI) is a prominent open-source software project that offers a complete ecosystem for building 4G and 5G networks. Initially developed by the Eurecom research institute, OAI now engages a global community of telecom experts and developers dedicated to advancing mobile network technology. OAI provides an end-to-end solution for mobile networks, including Evolved Packet Core (EPC) for 4G and Next Generation Core (NGC) for 5G, and can be configured for both research and real-world deployment.

Architecture and Components

OAI’s modular design includes components for both LTE and NR architectures. The platform provides:

  • Core Network (EPC & NGC): The core network manages user authentication, data routing, and connection management. OAI’s core components align closely with 3GPP standards, enabling interoperability with commercial LTE and NR devices.
  • Radio Access Network (RAN): The RAN module supports both LTE eNodeB and 5G gNodeB. It allows seamless integration with SDRs for real-time transmission and offers configurable parameters to customize network operations.
  • User Equipment (UE): OAI includes emulation capabilities for user devices, allowing users to test network configurations without physical devices. This is essential for labs and remote testing setups.

Prerequisites

  • Operating System: Linux (Ubuntu 20.04+ is recommended for compatibility)
  • Hardware: Compatible SDRs such as USRP B210 or N310 for live testing
  • Knowledge Requirements: Familiarity with cellular network concepts, basic Linux command-line proficiency, and network protocols
  • Software Dependencies: GCC, CMake, Flex, Bison, and additional network libraries

Key Capabilities

OAI enables users to perform a wide range of network research and testing tasks:

  • 4G/5G Deployment: Users can set up both standalone LTE and 5G networks, or integrate OAI with commercial networks for advanced testing scenarios.
  • Protocol Testing: OAI is frequently used for protocol validation, especially in academic research focused on optimizing LTE and NR protocols.
  • Interoperability Testing: Due to its adherence to 3GPP standards, OAI can be used to test interoperability with other telecom equipment and software, which is essential for commercial trials.
  • Experimentation with 5G Features: OAI supports new 5G capabilities such as network slicing, URLLC (Ultra-Reliable Low Latency Communication), and eMBB (Enhanced Mobile Broadband), making it ideal for cutting-edge 5G research.

srsRAN

Project Overview: Formerly known as srsLTE, srsRAN is an open-source, highly modular platform designed to provide a flexible and accessible LTE/5G software suite. Developed by Software Radio Systems (SRS), srsRAN is widely adopted in academia and industry for its simplicity and efficiency, especially for lightweight setups and lab environments. Unlike OAI, srsRAN offers a minimalist approach, with a primary focus on usability and modularity, making it an excellent choice for experimental deployments.

Architecture and Components

srsRAN includes a full suite of LTE and 5G components:

  • srsRAN eNodeB/gNodeB: These components handle radio access for LTE and NR networks. They support integration with a wide range of SDRs, providing users with real-time testing capabilities.
  • srsRAN EPC: The Evolved Packet Core (EPC) is fully integrated and manages core functions like mobility, session management, and authentication for LTE networks.
  • srsRAN UE: The user equipment (UE) module can be configured to emulate mobile devices, supporting LTE and limited 5G capabilities, making it a valuable tool for network performance assessment and protocol validation.

Prerequisites

  • Operating System: Linux (Ubuntu 18.04+ is recommended)
  • Hardware: SDRs compatible with srsRAN include USRP, LimeSDR, and BladeRF
  • Knowledge Requirements: Understanding of basic LTE/5G protocols, Linux command-line operations, and network analysis tools
  • Software Dependencies: GNU Radio, CMake, and GCC

Key Capabilities

srsRAN is known for its versatility and ease of use, supporting various use cases:

  • Standalone LTE Deployments: srsRAN’s lightweight design makes it ideal for rapid LTE deployment on low-cost SDR hardware, perfect for educational labs or experimental projects.
  • Lab and Field Testing: With its emulated UE and flexible configuration options, srsRAN allows users to test network configurations, experiment with different setups, and perform real-time over-the-air testing.
  • 5G Experimentation: srsRAN has expanded its support for 5G NR, including core and RAN elements. While not as feature-complete as OAI, it supports fundamental 5G research and offers a simpler alternative for early-stage 5G deployments.
  • Integration with SDR Platforms: srsRAN’s compatibility with a variety of SDRs makes it accessible for budget-conscious users, enabling extensive experimentation with both LTE and 5G configurations in real environments.

Open5GS

Project Overview: Open5GS is an open-source project that provides a complete 5G core network solution. It supports all the core network functions for 5G, including subscriber management, data routing, mobility management, and more. Open5GS is widely adopted for testing and research due to its flexibility and ease of use, providing a complete yet lightweight alternative for 5G core network deployments.

Architecture and Components

  • Core Network: Open5GS provides the full 5G core (5GC), including the AMF, SMF, UPF, and more, allowing full network operation.
  • Subscriber Management: Open5GS includes support for managing subscriber data, helping to maintain user profiles and provide network access controls.
  • Mobile Management: The project includes various components to manage mobility and session control for seamless communication.

Prerequisites

  • Operating System: Linux (Ubuntu 18.04 or 20.04)
  • Hardware: Server or virtualized environment capable of running the core network
  • Knowledge Requirements: Familiarity with core network principles, IP routing, and 5G architecture
  • Software Dependencies: Docker, MySQL/MariaDB, OpenSSL

Key Capabilities

  • 5G Core Network: Open5GS allows for testing and research on the entire 5G core network, supporting use cases such as private network deployments and integration with other platforms.
  • Interoperability: Open5GS offers extensive interoperability with commercial 5G devices, enabling real-world testing and trials.
  • Flexibility: Open5GS provides the ability to scale from small test environments to larger deployments.

Sonica NBIoT

Project Overview: Sonica NBIoT is an open-source platform that provides a framework for building and testing Narrowband IoT (NB-IoT) solutions. Designed for efficient communication in low-power, wide-area networks (LPWANs), Sonica NBIoT supports IoT applications and research in the context of 5G and beyond. It enables users to explore network protocols, optimize performance, and experiment with real-world NB-IoT scenarios.

Architecture and Components

  • NB-IoT Stack: Sonica NBIoT includes a complete software stack for NB-IoT, enabling communication between low-power IoT devices and the core network.
  • End-to-End Solution: It provides both the device and network components to facilitate end-to-end testing of NB-IoT services.

Prerequisites

  • Operating System: Linux (Ubuntu or CentOS)
  • Hardware: Compatible SDRs or NB-IoT modems
  • Knowledge Requirements: Basic knowledge of LPWAN and NB-IoT communication
  • Software Dependencies: Docker, OpenSSL, and specific libraries for IoT communication

Key Capabilities

  • Low-Power IoT Communication: Sonica NBIoT enables research and experimentation with LPWANs, providing insights into IoT network optimizations.
  • Integration with 5G: Sonica NBIoT supports integration with 5G networks, allowing users to explore the role of NB-IoT in 5G ecosystems.
  • Flexible Testing: The platform offers flexible configurations to test various NB-IoT deployment scenarios, from rural areas to urban settings.
